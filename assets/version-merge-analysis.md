# Version Merge Analysis

## Marker Mapping

- Marker 3: current personal `solution-factory` plugin workspace; strongest at multi-agent decomposition, smart-logistics pilot artifacts, style prompts, NotebookLM handoff, and Codex plugin packaging.
- Marker 2: local `解决方案部 AI PPT 生产线`; strongest at browser workbench, upload intake, requester workspace isolation, company knowledge-base management, page-level script workbench, and no-placeholder startup state.
- Marker 1: company Linux server package `solution-ppt-script-workbench`; final reference for production boundaries, Hermes/Codex model adapters, server deployment scripts, quality gates, and teammate-safe capability copy.

## Merge Decision

The unified plugin keeps marker 3's Codex plugin shape, marker 2's job/workspace artifact contract, and marker 1's server production boundary.

The canonical first production path is script and image-prompt generation. Image PPT rendering and editable PPT rebuild remain downstream reviewed stages, not default server claims.

## Bug Fixes Absorbed

- New jobs must not generate placeholder page scripts or sample pages by default.
- Stage 2 image generation must not be shown as connected unless a real executor writes formal assets.
- Local HTML/SVG/screenshots may be preview/debug artifacts only, never formal image PPT output unless explicitly approved as that route.
- Page count is strict, including cover and agenda.
- P01/P02 roles are fixed.
- Weak page titles and duplicate titles are rejected.
- Every page must include `Page-specific source of truth`.
- Company knowledge base cannot replace customer material facts.
- Model/provider names must be honest: Codex, Hermes, OpenAI API, DeepSeek, or other adapters appear only after verification.
- Server/browser users must not see shell paths, model credentials, or arbitrary local file access.
