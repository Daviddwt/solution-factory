# Operator Intake Checklist

Use this checklist at the start of every project.

## Required Project Inputs

- Project/customer name
- Industry/domain
- Customer background
- Proposal purpose
- Target audience
- Desired output format
- Deadline or milestone
- Language and tone
- Required page count or depth, if any

## Required Material Buckets

1. Customer materials
   - RFP, requirement list, meeting minutes, screenshots, existing PPT, architecture diagrams, process documents, emails, questionnaires.

2. Existing environment
   - Existing systems, APIs, databases, devices, network boundaries, user portals, enterprise IM, approval systems, reporting tools.

3. Company capability baseline
   - Product FRS, product manuals, capability lists, standard solution decks, case studies, company profile.

4. Delivery and commercial boundaries
   - Implementation constraints, security requirements, data policies, deployment rules, service boundaries.
   - Store only summaries and source paths for sensitive commercial materials.

5. PPT style references
   - Reference PPTX, PDF, screenshots, corporate templates, customer-branded decks, or previous approved decks.

## Missing-Material Handling

If a material bucket is missing:

- Ask the operator to upload it or confirm it is unavailable.
- Mark the gap in `open_questions`.
- Do not invent capabilities or customer facts.
- If the operator asks to proceed anyway, label the related output as assumption-based.

## Operator Approval Summary

Before production, summarize:

- Materials received
- Materials missing
- Approved source folders/files
- Sensitive handling rules
- PPT style source
- Expected artifacts
- Review gates
- Known assumptions
